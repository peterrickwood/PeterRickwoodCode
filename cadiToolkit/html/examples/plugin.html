Writing a plugin for the CADI toolkit

All plug-ins for the CADI tookkit must be distributed as Java archives (.jar files). In practice, this means that they must be written in Java.[1].

All classes in the archive must be in their own package (i.e. they cannot be in a 'default' package).

The jar file must contain a Manifest with the Main-Class attribute specifying the class that is to be 'run' if the extension is selected in the toolkit. This main class must either extend java.lang.Thread or implement java.lang.Runnable. The preference is that you implement java.lang.Runnable. The main class must also have a no-argument constructor.

You must place this jar file in a directory called 'ext' in the cadi client directory. For example, if your cadi client is installed in /some/path/cadiclient then your plugin must be placed at /some/path/cadiclient/ext/myplugin.jar.

The extension will be added to the "Miscellaneous" menu in the toolkits main menubar. If selected, the zero argument constructor of the main class is called, and then the class' run() method is set running in a separate thread.

[1] Or a language that can compile to Java bytecode. Python, for example can be compiled to java bytecode by the Jython python compiler/interpreter.
